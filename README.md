# TGCrypto Bot (воронка с проверкой подписки)

Бот делает сценарий:

- `/start` → приветственное медиа + текст + кнопка **«Забрать уроки»**
- описание 3 уроков + кнопка-ссылка на канал + **«✅ Я подписался»**
- если подписка есть → выдача 3 видеоуроков по очереди, после каждого кнопка **«Я посмотрел»**

## Требования

- Node.js 20+ (рекомендуется)
- Бот **должен быть админом** канала, чтобы проверка подписки через `getChatMember` работала надёжно

## Быстрый старт

1) Установить зависимости:

```bash
npm i
```

2) Скопировать и заполнить env:

```bash
cp env.example .env
```

3) Запуск в dev-режиме:

```bash
npm run dev
```

## Деплой / обновление на VPS

В репозитории есть готовый вариант автодеплоя через `git push` (bare repo + `post-receive` hook) и запуск через `systemd`:

- инструкции: `deploy/SETUP.md`
- systemd unit: `deploy/tgcrypto.service`
- hook: `deploy/post-receive`

После того как вы один раз настроили сервер по `deploy/SETUP.md`, **для обновления бота на VPS достаточно пуша**:

```bash
git push vps main
```

Хук на сервере автоматически:

- обновит рабочую копию (`git checkout`)
- выполнит `npm ci`
- соберёт проект (`npm run build`)
- перезапустит сервис `tgcrypto` (`systemctl restart tgcrypto`)

Проверка логов на сервере:

```bash
journalctl -u tgcrypto -n 50 --no-pager
```

## Как получить `file_id` для фото/видео

Самый простой способ:

1) Отправьте нужный файл вашему боту в личку.
2) Включите режим `DEBUG_FILE_ID=1` в `.env` и перезапустите бота.
3) Бот ответит `photo.file_id: ...` или `video.file_id: ...`.

В `.env` можно использовать **и URL**, и `file_id`. `file_id` обычно быстрее и стабильнее.

## Сброс состояния при тестах

Телеграм “не очищает чат” программно для бота, но для прогона сценария заново достаточно сбросить состояние в SQLite.

- **Сбросить только себя**: `/reset`
- **Сбросить всех**: `/reset_all` (доступно только id из `ADMIN_IDS` в `.env`)
- **Самый быстрый полный сброс**: остановить бота и удалить файл `data/bot.sqlite`, затем запустить снова

## Структура проекта

- `src/index.ts` — старт бота
- `src/flows/funnel.ts` — логика сценария и кнопки
- `src/telegram/subscription.ts` — проверка подписки на канал
- `src/db/*` — SQLite и репозиторий пользователей
- `data/bot.sqlite` — файл базы (создаётся автоматически)
